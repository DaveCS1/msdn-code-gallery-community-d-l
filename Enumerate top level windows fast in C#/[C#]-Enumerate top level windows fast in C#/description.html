Introduction

A poster at the MSDN Forums asked (eventually) for a way to enumerate top-level windows in C#.  I created this quick sample to provide a simple solution to the problem.  I hope that it is helpful to more than one.

Description

The problem of enumerating the windows is really not much of a problem.  The Windows API EnumWindows() can do this just fine.  But for some people it is difficult to grasp P/Invoke and it is hard for them to translate a C declaration into a useful .Net/C# one, especially if it involves function callbacks, such as this case.

The solution sample shows the use of a delegate (as they serve as function pointers) to enumerate all top-level windows that have a title and that are visible.  Visibility is determined by a call to the Windows API function IsWindowVisible(), and has no regard as to whether the window is actually visible on screen, as it may be behind another window.

The Code

The core functionality is really provided by Windows in the form of the Windows API functions.  This is how they are brought into play in the C# world.  Note the first item:  It is the delegate declaration for the callback procedure that is required for the EnumWindows() function to be of any good use.  The rest are just imports of the required Windows API functions.

The sample code uses GetWindowTextLength() and GetWindowText() to read the window's title.

        protected static bool EnumTheWindows(IntPtr hWnd, IntPtr lParam)
        {
            int size = GetWindowTextLength(hWnd);
            if (size++ > 0 && IsWindowVisible(hWnd))
            {
                StringBuilder sb = new StringBuilder(size);
                GetWindowText(hWnd, sb, size);
                Console.WriteLine(sb.ToString());
            }
            return true;
        }
